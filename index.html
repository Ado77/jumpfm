About

JumpFm is a minimalistic dual panel file manager for Linux that lets you navigate your file system very efficiently. It is highly configurable and extendible and it comes with some builtin super-powers.

Jumping

Press j to jump directly to your favorites folders. No manual bookmarking is needed, JumpFm will learn where you want to go.

Git Awareness

Files are displayed according to thier git status.

Filtering

Press f to quickly find the files you are looking for.

Flat Mode

Press r and all directories are gone. Press it again to bring them back.

Instant Gist

Press ctrl + g to create instant gist from selected files.

External Programs

Configure hot-keys to run external tools.

Before You Jump

Instant gists are public !!! don't gist your private ssh key or any other private information.

There are no confirmation dialogs in JumpFm, whenever you hit the DEL key a file is gone.

Settings

All settings files are located under the ~/.jumpfm directory. You can edit the settings files to make JumpFm your own. You can also safely delete the settings folder and restart JumpFm to restore the default settings.

Default Settings:

Key Default Value Description
jumpMaxDbSize 300 Set the maximum number of folders that JumpFm index for fast jumping
jumpDbSaveInterval 5 Controls how often JumpFm update the jumps database
historyMaxSize 20 Set the maximum history stack size
flatModeMaxSize 500 Set the maximum number of files the flat mode can display
dateFormat MM/DD/YYYY hh:mm:ss Sets the clock format
editor gedit Default editor to edit files
weatherLocation Haifa, IL Default weather location

Plugins

Plugins control every aspect of JumpFm. You can enable/disable them, install new once and also write your own.

Builtin plugins:

Plugin Enabled Description Safely Disable
version true Displays the current version of JumpFm. yes
clock true Displays the date and time. yes
weather true Displays local (need to set the weatherLocation value) weather yes
file system true List the files in the directory. no
key navigation true Enable keyboard shortcuts. no
file operations true Enable file operations such as renaming, editing, etc... no
copy true Enable file copying. no
jump true Enable directory jumping. yes (but why?)
history true Enable history navigation. yes (but why?)
flat mode true Enable flat mode. yes
zip true Enable zipping and unzipping files. yes
git status true Enable git status awareness. Might cause some performance degradation when browsing large git repositories. yes
gist true Allow sharing files via public gists. If you decide to enable this plugin make sure you know what public means. yes

External Programs

You can bind hotkeys to trigger the execution of external applications. You can pass those applications arguments based on the current state.

Available Variables:

Variable Description
{active_pwd} The full path of the active panel
{active_cur} The file under cursor on the active panel
{active_selected} The list of selected files in the active panel
{passive_pwd} The full path of the active panel
{passive_cur} The file under cursor on the active panel

Examples:

git difftool {selected} will trigger the diff tool configured for git with the selected files passed as arguments to the program.

git difftool {active} will trigger the diff tool configured for git with the full path of the active panel passed as an argument to the program.
